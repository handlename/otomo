# Otomo MCP Integration Skill

This skill allows the agent to communicate with the active `otomo chat` session running on the local machine.

## How it works
When the user runs `otomo chat --mcp`, a local MCP server (SSE mode) is started at `http://localhost:8000/sse` (or custom port).
By connecting to this endpoint, the agent can access the `post_message` tool.

## Guidelines for using the tool
- **When to use**: Use this skill when you want to report progress, request user clarification, or log important events directly to the user's terminal chat screen.
- **Protocol**: 
  - Host: `http://localhost:8000` (default)
  - Endpoint: `/sse`
  - Protocol: Model Context Protocol (SSE transport)
- **Available Tools**:
  - `post_message(message: string)`: Sends a message to the active chat screen. This message will be displayed as coming from the "Agent". The agent will receive the output response generated by `otomo` after processing the message.

## Example MCP Client Configuration
```json
{
  "mcpServers": {
    "otomo": {
      "type": "sse",
      "url": "http://localhost:8000/sse"
    }
  }
}
```
