# GLOSSARY

This glossary defines key terms, domain concepts, and technical terminology used across the Otomo project. It serves as a reference for developers implementing the bot and understanding its architecture.

## Domain & AI Concepts

### Answer

The response content generated by the [Brain](#brain).

### Brain

An abstraction representing the AI reasoning and response generation capability, typically implemented using [AWS Bedrock](#aws-bedrock).

### Context

The conversational state and history maintained during an interaction with a user.

### InstructionReceived

A domain event published to trigger asynchronous processing when a user sends an instruction to the bot.

### Otomo

The core domain interface and orchestrator representing the AI agent itself, coordinating the [Brain](#brain) to generate replies to instructions.

### Prompt

A domain value object representing the constructed prompt template sent to the [Brain](#brain).

### Reply

A domain entity representing the bot's response logic, which eventually delivers an [Answer](#answer) to the user.

### Thread

A representation of a Slack conversation thread.

### ThreadMessage

An individual message within a [Thread](#thread).

### Tool

A function or capability registered and provided to the AI [Brain](#brain) to perform specific actions.

## Architecture & Technology

### Application Layer

The layer responsible for coordinating use cases, managing services (such as [Messenger](#messenger)), and orchestrating the flow of data.

### AWS Bedrock

The managed cloud service by AWS used as the implementation for the AI [Brain](#brain).

### Domain Layer

The core of the system containing business entities (e.g., [Brain](#brain), [Thread](#thread)) and repository interfaces, completely independent of external libraries or frameworks.

### Infrastructure Layer

The outermost layer that implements adapters for external systems, such as [AWS Bedrock](#aws-bedrock), Slack APIs, and storage.

### Messenger

An abstraction interface for sending messages to a chat platform (like Slack), defined in the [Application Layer](#application-layer).

### Presentation Layer

The entry points of the application, responsible for routing HTTP requests (Slack events), handling CLI commands, and execution on AWS Lambda.

### Ridge

A framework that adapts standard Go HTTP routers (such as chi) to work seamlessly in an AWS Lambda environment.
